# DVS Viewer 用户手册

> 适用硬件：Prophesee IMX636 / EVK4 DVS 相机
> 软件版本：GitHub `marui-tech/dvs-viewer`

---

## 一、界面总览

程序启动后窗口分为 **四列控制面板 + 中央渲染区** 两部分。

```
┌─────────────┬──────────────────┬─────────────────────┬──────────────────────────────┬─────────────────────┐
│  Col 1      │  Col 2           │  中央渲染区          │  Col 3                       │  Col 4              │
│  相机/状态  │  可视化/偏置     │  事件画面            │  硬件控制                    │  触发/监控/录制     │
└─────────────┴──────────────────┴─────────────────────┴──────────────────────────────┴─────────────────────┘
```

每列顶部有 **◀ 按钮**，点击可以折叠列，给渲染区域腾出空间。
每列内部各功能区也可以点击标题行**折叠/展开**。

---

## 二、基本工作流程

### 第一步：连接相机

1. 将 EVK4 用 USB 3.0 连接到电脑
2. 点击 **Col 1 → ⚡ 连接相机 Connect**
3. 等待 2–5 秒，连接成功后状态栏显示 `Connected`，序列号和分辨率自动填写

> 如果连接失败，检查设备管理器中 EVK4 是否有 WinUSB 驱动（见 `drivers/README.md`）

### 第二步：开始采集

点击 **▶ 开始采集 Start**，中央区域出现实时事件流画面：
- **绿点** = ON 事件（亮度增加）
- **红点** = OFF 事件（亮度减少）

### 第三步：停止/断开

- **■ 停止采集 Stop**：停止事件流，保持连接
- **⏏ 断开 Disconnect**：完全断开，释放设备

## 三、Col 1 — 相机 / Camera

### 实时统计

| 字段 | 含义 |
|------|------|
| 状态 State | 当前相机状态（Disconnected / Connected / Streaming / Recording） |
| 事件率 Event Rate | 每秒事件数（kev/s 或 Mev/s） |
| 总量 Total | 本次采集累计事件数 |
| 录制 Recording | 当前录制状态和文件名 |

### 渲染性能

| 字段 | 含义 | 正常范围 |
|------|------|---------|
| FPS | 渲染帧率 | 400–900 fps（无事件时） |
| 帧耗时 Paint | 单帧 GPU 渲染时间 | < 2 ms |
| 等待耗时 Wait | 事件到达前的等待时间 | — |
| 事件/帧 Evs/frm | 每帧平均渲染事件数 | — |
| 包数/帧 Batches | 每帧合并的事件包数量 | 1–10 |
| VBO带宽 VBO BW | 显存上传带宽 | — |

> **FPS 下降说明**：事件率越高，渲染 GPU 负担越大。
> 20 Mev/s 时 FPS 约 200，属正常现象，不影响事件完整性。

## 四、Col 2 — 可视化/偏置

### 4.1 可视化设置

#### 模式选择

| 模式 | 效果 | 适用场景 |
|------|------|---------|
| **event_frame** | ON=绿 / OFF=红，有衰减拖尾 | 日常观察，默认推荐 |
| **time_surface** | 像素颜色表示最近一次事件的时间（年龄热图） | 观察运动轨迹 |
| **on_only** | 只显示 ON（绿）事件 | 观察亮区边缘 |
| **off_only** | 只显示 OFF（红）事件 | 观察暗区边缘 |
| **accumulated** | 不衰减，事件永久叠加 | 分析稀疏场景 |
| **instant** | 每帧完全清除，只显示当前 Δt 切片，无任何拖尾 | **超高速目标**，配合 Δt=1–2ms 使用 |

#### 衰减系数 Decay Factor

- 范围 0.01–0.99，越大拖尾越长
- `0.72` 为默认，适合中等速度场景
- 高速物体建议调低（0.3–0.5），慢速场景建议调高（0.85–0.95）
- `accumulated` 模式下无效

#### 事件切片 Δt (µs)

- 控制相机每次采集的时间窗口长度，默认 10,000 µs = 10 ms
- 调小（如 1000 µs）可提高时间分辨率，适合分析高速运动细节
- **修改后需重启采集（Stop → Start）才生效**

#### 清空帧缓冲

点击 **🗑 清空帧缓冲** 立即清屏，不影响采集。切换场景或模式后使用。

---

### 4.2 偏置调节（Bias Tuning）

> 偏置是像素灵敏度相关的传感器寄存器参数，调整前建议先用**快速预设**。

#### 快速预设（推荐先用）

| 预设 | 适用场景 | 自动设置 |
|------|---------|---------|
| **默认** | 室内通用 | ERC 5 Mev/s，关闭活动滤波 |
| **高速目标** | 快速运动物体（< 1ms 精度） | ERC 20 Mev/s，活动滤波上限 500K ev/s/px |
| **低噪声** | 慢速静态场景，抑制背景噪声 | ERC 2 Mev/s，活动滤波 50–2M ev/s/px |

#### 各偏置参数含义

| 参数 | 含义 | 建议范围 |
|------|------|---------|
| `bias_diff_on` | ON 事件触发阈值（越小越灵敏） | 88–130 |
| `bias_diff` | 基准对比度阈值 | 72–90 |
| `bias_diff_off` | OFF 事件触发阈值 | 44–90 |
| `bias_refr` | 像素不应期（越大越慢，抑制热像素） | 0–80 |
| `bias_fo` | 低通截止频率 | 72（一般不调） |
| `bias_hpf` | 高通截止频率 | 0–20 |

> **IMX636 重要约束**：
> `diff_on - diff ≥ 15`，`diff - diff_off ≥ 10`
> 违反此约束会导致像素不稳定，事件率降到 < 1 kev/s。

## 五、Col 3 — 硬件控制

### 5.1 ERC 事件率控制

**作用**：在传感器内部限制最大输出事件率，超过阈值的事件在芯片内丢弃。

| 参数 | 说明 |
|------|------|
| 启用 Enable | 勾选后才生效 |
| 阈值 Threshold | 单位 events/s，默认 5,000,000（5 Mev/s） |

**使用场景**：
- 场景太亮、事件过多导致画面白屏 → 调低阈值（如 5M）
- 高速拍摄需要更多细节 → 调高至 20M，但同时开活动滤波抑制热像素

点击 **应用 Apply ERC** 立即生效，无需重启。

---

### 5.2 ROI 感兴趣区域

**作用**：在传感器级别只开启指定矩形区域内的像素，其余区域的像素完全关闭。

| 参数 | 说明 |
|------|------|
| X, Y | 矩形左上角坐标（像素） |
| 宽 Width | 矩形宽度 |
| 高 Height | 矩形高度 |

**使用场景**：只关注画面中央某个物体，大幅降低事件率和录制文件大小。

---

### 5.3 抗闪烁 Anti-Flicker

**作用**：过滤由电源频率（50Hz/60Hz）引起的周期性噪声事件。

- 室内日光灯/LED 场景建议启用
- 设置下限/上限覆盖当地电源频率（中国 50Hz，设为 45–55 Hz 即可）

---

### 5.4 活动滤波 Activity Filter

**作用**：按每个像素的事件产生速率进行门控，速率在 [下限, 上限] 区间外的像素被抑制。

| 参数 | 说明 |
|------|------|
| 下限 Lower | 最低事件率（ev/s/px），低于此值的像素被抑制（过滤静态噪声） |
| 上限 Upper | 最高事件率（ev/s/px），高于此值的像素被抑制（过滤热像素） |

**热像素问题**：当 `bias_refr=0`（高速预设）时，部分像素会以 MHz 级频率持续触发，形成方块状噪声。
设置上限 500,000 ev/s/px 可有效抑制（热像素 >>500K，正常运动事件 << 500K）。

---

## 六、Col 4 — 触发/监控/录制

### 6.1 触发输入 Trigger In

**作用**：接收外部电气脉冲信号，记录精确时间戳用于多设备时间同步。

| 参数 | 说明 |
|------|------|
| 通道 Channel | 选择触发通道（0–7，IMX636 通常只有通道 0） |

点击 **启用 Enable** 后，外部脉冲会在事件流中产生 `EventExtTrigger` 事件。

---

### 6.2 触发输出 Trigger Out

**作用**：相机向外部设备输出周期性脉冲，用于驱动高速相机、激光等同步。

| 参数 | 说明 |
|------|------|
| 周期 Period (µs) | 脉冲周期，默认 1,000,000 µs = 1 Hz |
| 占空比 Duty Cycle | 高电平时间占比，0.5 = 50% |

---

### 6.3 传感器监控

显示传感器实时温度（°C）和光照（lux）。
勾选 **自动刷新 Auto-refresh** 每 2 秒更新一次。

---

### 6.4 录制

#### 基本录制步骤

1. 在 **路径 Path** 中填写文件名（如 `test.npy`），或使用绝对路径（如 `D:/data/run1.npy`）
2. 点击 **⏺ 开始录制 Start Recording**
3. 录制完成后点击 **⏹ 停止录制 Stop Recording**
4. 等待界面显示 `✅ 已保存 ...` 提示

文件保存为 NumPy `.npy` 格式，可用 Python 读取：

```python
import numpy as np
evs = np.load('recording.npy', allow_pickle=True)
# evs 字段：x (uint16), y (uint16), p (int16, 0=OFF/1=ON), t (int64, 微秒)
print(evs[:5])
```

#### 录制 ROI 过滤

勾选 **区域过滤 ROI Filter** 后，只录制指定矩形内的事件：
- `X0, Y0`：矩形左上角
- `X1, Y1`：矩形右下角
- 点击 **↑ 同步相机 ROI**：自动填入上方硬件 ROI 的参数

**使用场景**：只关注画面局部，大幅减小文件体积，延长有效录制时间。

#### 录制密度

| 选项 | 说明 |
|------|------|
| 全量 ×1 | 保存所有事件（默认） |
| 1/2 | 每 2 个事件保存 1 个 |
| 1/4 | 每 4 个事件保存 1 个 |
| 1/8 | 文件缩小 8 倍 |
| 1/16 | 文件缩小 16 倍，适合超长录制 |

**使用场景**：高速场景下事件密集，用 1/4 或 1/8 可在保持运动可见性的同时大幅缩小文件。

---

### 6.5 回放

#### 打开并播放文件

1. 点击 **📂 选择文件** 选择一个 `.npy` 录制文件
2. 如果录制时使用了非 1280×720 的传感器或 ROI，修改 **传感器 W / H**
3. 调整 **回放速度 Playback Speed** 滑块
4. 点击 **▶ 载入并播放 Load & Play**

播放开始后，窗口底部出现 **播放控制栏**：

```
[进度条（可拖动）]
⏮  ▶  ⏹   0:05 / 1:23   × 0.5
```

| 按钮 | 功能 |
|------|------|
| ⏮ | 回到开头 |
| ▶ / ⏸ | 播放 / 暂停 |
| ⏹ | 停止并关闭播放栏 |

#### 速度控制

速度滑块使用**对数刻度**：

| 滑块位置 | 速度 | 适用 |
|---------|------|------|
| 0（最左） | × 0.0001 | 逐帧级慢放，看单个事件 |
| 约中间 | × 1.0 | 实时速度 |
| 100（最右）| × 10 | 快速预览 |

## 七、常见问题

### Q：连接后事件率显示 0.8 kev/s，场景明明有运动

**原因**：偏置参数不合理，`diff_on - diff` 间距太小（< 15）导致 IMX636 像素不稳定。
**解决**：点击偏置区 **↺ 恢复默认 Reset Defaults**，或点击 **默认** 预设。

---

### Q：高速预设切换后画面全白

**原因**：ERC 被完全禁用，66 Mev/s 的事件流淹没渲染器。
**解决**：软件已修复，高速预设会自动设置 ERC 20 Mev/s。
如仍出现，手动在 Col 3 → ERC 中勾选启用并设置 5–20 Mev/s。

---

### Q：回放时有大量红绿散点噪声（方块/短横线）

**原因**：录制时 `bias_refr=0`（高速预设），热像素以 MHz 频率触发，这些事件已混入录制文件。
**解决（录制前预防）**：
- 采集时开启 **活动滤波 Activity Filter**，上限设为 500,000 ev/s/px
- 或使用 **高速目标** 预设（会自动开活动滤波）

回放已有噪声的文件目前没有软件滤波功能，建议重新录制。

---

### Q：FPS 从 800 下降到 200

**原因**：事件率高时 GPU 渲染负担增大，属正常现象。
事件完整性不受影响，每帧只是渲染了更多事件。
如需保持高 FPS，可降低 ERC 阈值减少事件量。

---

### Q：录制很久后停止卡顿

**原因**：旧版程序在停止时需要将临时文件转换为 .npy，大文件耗时较长。
当前版本已优化为流式 IO，停止时只需等待很短时间（通常 < 3 秒）。

---

### Q：主程序连接时卡顿几秒

**原因**：相机初始化在后台线程执行，界面会短暂无响应。
这是正常现象，等待状态变为 `Connected` 即可。

## 八、快捷操作建议

| 目标 | 推荐操作 |
|------|---------|
| 室内日常观察 | 默认预设 + event_frame 模式 + Decay 0.72 |
| 拍高速运动 | 高速目标预设 → 自动设置 ERC+活动滤波 |
| 抑制背景噪声 | 低噪声预设 |
| 只录制画面局部 | 录制 ROI 过滤 + 同步相机 ROI |
| 长时间录制 | 录制 ROI + 密度 1/4 或 1/8 |
| 分析单帧细节 | 回放 + 速度调到 × 0.001 慢放 |
| 清除残影 | 🗑 清空帧缓冲 |
